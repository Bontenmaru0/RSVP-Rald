Architecture Notes
- Feature-first vertical slicing:
  - Each domain keeps `data`, `state`, and `presentation` together.
- Riverpod state flow:
  - UI watches providers in `state/`.
  - Controllers orchestrate async operations and loading/error states.
  - Services in `data/` isolate Supabase RPC/storage/auth calls.
- Comments domain design:
  - `CommentsState` stores lists in maps keyed by `articleId` and `imageId`.
  - Per-comment loading maps support granular update/delete indicators.
- Backend integration:
  - Supabase is initialized at app startup from `.env` keys.
  - Features call RPC functions and storage buckets through service classes.
